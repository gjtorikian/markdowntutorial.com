Liens

Nous allons maintenant apprendre comment créer des liens vers d'autres sites Web sur le Web.

Il y a deux types de liens différents dans Markdown, mais les deux ont exactement un rendu identique. Le premier style de lien s'appelle un lien in line (ou dans la ligne) . Pour créer un fichier en ligne vous encadrez le texte du lien entre crochets ( [ ] ), puis vous encadrez le lien entre parenthèses ( ( ) ). Par exemple, pour créer un lien hypertexte vers www.github.com, avec un texte de lien qui dit : "Visitez GitHub !", vous écrirez ça en Markdown : [Visitez GitHub !](www.github.com).

L'autre type de lien s'appelle un lien référence. Comme son nom l'indique, le lien est en fait une référence à un autre endroit dans les documents. Voici un exemple de ce que nous voulons dire :

    Voici [un lien vers autre chose][un autre endroit].
    Voici [encore autre lien][un autre lien].
    Et maintenant, revenons au [premier lien][un autre endroit].

    [un autre endroit] : www.github.com
    [autre lien] : www.google.com

Les "références" ci-dessus constituent le deuxième ensemble de crochets : [autre endroit] et [autre-lien]. Au bas d'un document de démarque, ces parenthèses sont définies comme suit en tant que liens appropriés vers des sites Web externes. Un avantage du style de lien de référence est que plusieurs liens vers le même endroit n'ont besoin d'être mis à jour qu'une seule fois. Pour exemple, si nous décidons de faire en sorte que tous les liens [autre endroit] aillent quelque part sinon, nous n'avons qu'à changer le lien de référence unique.

Les liens de référence n'apparaissent pas dans le rendu Markdown. Vous les définissez en fournissant le même nom de balise entre parenthèses, suivi d'un deux-points, suivi du lien.

Dans l'encadré ci-dessous, nous avons commencé à écrire quelques liens de référence. Vous voyez, ils ont besoin d'être fini ! Appeler la première balise de référence "un endroit amusant", et faire le lien vers www.zombo.com ; faire le second lien vers www.stumbleupon.com.

Vous savez maintenant comment faire des liens avec le Markdown !
